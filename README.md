# kf-css

**A modern, efficient CSS framework tailored for SvelteKit.**  
_Lightweight. Semantic. Customizable._

## üöÄ Features

- **SvelteKit First**: Designed with SvelteKit's architecture in mind.
- **Automated**: Custom Vite plugin updates styles instantly as you edit.
- **Responsive**: Mirror utility that automatically generates responsive variants (e.g. `.m:p-4`).
- **Semantic**: Built on a solid design system of colors, typography, and spacing.
- **Customizable**: Built with Sass, easily configured via variables.

## üì¶ Installation & Setup

**Step 1: Install the Package**

```bash
npm install kf-css
```

**Step 2: Scaffold the Source**
Run the init command. This copies the full Sass source and build scripts into your project (default: `src/lib/kf-css`), giving you full ownership.

```bash
npx kf-css
```

**Step 3: Install Sass Compiler**
You need Sass to compile the source files.

```bash
npm install -D sass
```

**Step 4: Configure SvelteKit**
Add the Vite plugin and import the CSS.

1.  **Update `vite.config.js`**:
    Import the `kfCss` plugin to enable auto-rebuilding of the responsive classes.

    ```javascript
    import { sveltekit } from "@sveltejs/kit/vite";
    import { defineConfig } from "vite";
    import { kfCss } from "./src/lib/kf-css/bin/vite-plugin.js";

    export default defineConfig({
      plugins: [sveltekit(), kfCss()],
    });
    ```

2.  **Import in Reference**:
    In `src/routes/+layout.svelte` (or your root file), import the **generated** responsive CSS.

    ```svelte
    <script>
      import '$lib/kf-css/kf-responsive.css';
    </script>

    <slot />
    ```

## ‚ö° Optimization

### PurgeCSS

**Note:** SvelteKit automatically minifies (compresses) your CSS for production. However, to **remove unused styles** (especially the many responsive utility classes generated by kf-css), you MUST use PurgeCSS.

1.  **Install PurgeCSS**:

    ```bash
    npm install -D @fullhuman/postcss-purgecss
    ```

2.  **Configure**:
    Point PurgeCSS to your source files so it knows what you are using.
    ```javascript
    content: ['./src/**/*.{html,js,svelte,ts}'],
    css: ['./src/lib/kf-css/dist/kf.css'] // + kf-responsive.css
    ```

## üé® Customization

The framework is configured via Sass variables in `src/config/`.

1.  **Colors**: Edit `config/colors.scss` to define your palette.
2.  **Typography**: Adjust `config/typography.scss` for font stacks and scales.
3.  **Breakpoints**: Modify `config/layout.scss`.

The `bin/mirror.js` script reads your CSS variables (specifically `--breakpoint-*`) to generate responsive classes.
